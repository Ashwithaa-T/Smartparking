# Smart Parking Availability Prediction

A comprehensive Smart Parking ML solution that uses Machine Learning to predict parking slot availability across various locations in India. It includes an ML model training pipeline, a FastAPI backend serving real-time predictions, and a React-based frontend.

## Overview

The system predicts whether parking slots are available based on time of day, day of the week, and historical patterns. If a slot is occupied, it estimates when it will become available next.

### Key Features
- **Machine Learning Predictive Engine**: Uses Logistic Regression to determine if a slot will be occupied, and a Random Forest Regressor to predict the expected duration of occupancy based on historical data.
- **FastAPI Backend**: A high-performance API that models virtual parking slots using the trained ML behavior templates for major locations across India (malls, airports, tourist spots).
- **React Frontend**: An interactive UI for users to check live availability and see individual predicted availability times for each parking slot.
- **Real-Time Simulation**: Uses probabilities generated by the model to simulate realistic slot-by-slot distribution.

## Project Structure

- `SPA_log.py`: The machine learning pipeline. It loads data, performs feature engineering, trains a Logistic Regression model for availability and a Random Forest model for duration, and exports both models.
- `main.py`: The FastAPI application. It loads the pre-trained models and exposes an endpoint (`/predict`) to query parking status by location.
- `SPSIRDATA.csv`: The dataset used to train the machine learning models.
- `availability_model.pkl`: The serialized Logistic Regression model for availability prediction.
- `duration_model.pkl`: The serialized Random Forest model for predicting occupied durations.
- `frontend/`: The directory containing the React frontend application.

## Technologies Used

- **Backend**: Python, FastAPI, Uvicorn, Pydantic
- **Machine Learning**: Pandas, NumPy, Scikit-Learn, Joblib, Matplotlib
- **Frontend**: React.js, TailwindCSS (assumed from the `.jsx` file)

## Setup and Installation

### Prerequisites
- Python 3.8+
- Node.js & npm (for frontend)

### 1. Clone the repository
```bash
git clone https://github.com/Ashwithaa-T/Smartparking.git
cd Smartparking
```

### 2. Backend Setup
Install the required Python dependencies:
```bash
pip install -r requirements.txt
```

*(Optional)* Retrain the ML models if you have new data:
```bash
python SPA_log.py
```
This will read `SPSIRDATA.csv`, train the models, show accuracy and confusion matrix, and save `availability_model.pkl` and `duration_model.pkl`.

### 3. Run the API Server
Start the FastAPI application using uvicorn:
```bash
python main.py
# OR
uvicorn main:app --host 127.0.0.1 --port 8001 --reload
```
The API will be available at `http://127.0.0.1:8001`. You can access the automatic interactive API documentation at `http://127.0.0.1:8001/docs`.

### 4. Frontend Setup
Navigate to the frontend directory and run the development server:
```bash
cd frontend
npm install
npm start # or npm run dev depending on setup
```

## API Endpoint Reference

### `POST /predict`
Predicts parking availability for a specific location.

**Request Body:**
```json
{
  "location_name": "Phoenix Palladium Mall",
  "hour": 14,
  "day": 2
}
```
*(Note: `hour` and `day` are optional. If omitted, the API uses the current system time).*

**Response:**
```json
{
  "location_name": "Phoenix Palladium Mall",
  "city": "Mumbai",
  "state": "Maharashtra",
  "location_type": "Mall",
  "total_slots": 320,
  "available_slots": 145,
  "occupied_slots": 175,
  "availability_percentage": 45.31,
  "status": "Available",
  "timestamp": "2026-06-06T16:41:04.123456",
  "prediction_method": "ML Templates",
  "estimated_time": "05:15 PM",
  "slots": [
    {
      "status": "occupied",
      "nextAvailable": "05:15 PM"
    },
    {
      "status": "available",
      "nextAvailable": null
    }
  ]
}
```
